# ProtectionStones — Custom Fork

Форк плагина [ProtectionStones 2.10.5](https://github.com/espidev/ProtectionStones) для Paper 1.21.10.

> **Совместимость:** Paper 1.21.10 · WorldGuard 7.0.15 · WorldEdit 7.4.2 · Vault 1.7.3

---

## Дополнительная функция: внутренние участки (`/ps plot`)

### Зачем это нужно

ProtectionStones защищает территорию через установку специального блока. Эта функция позволяет **разбить уже защищённую территорию на именованные участки** без установки дополнительных блоков — просто выделяешь зону топориком WorldEdit и создаёшь участок.

Типичный сценарий — клановая территория: один игрок ставит блок и защищает большую зону, а затем раздаёт членам клана отдельные личные участки внутри неё. Каждый работает только на своём участке, не имея доступа к остальной территории.

---

## Команды

Все команды работают из любой точки мира. Аргументы с именем участка поддерживают автодополнение по Tab.

### `/ps plot create [название]`

Создаёт участок из текущего выделения деревянным топориком WorldEdit.

**Порядок действий:**
1. Выдели зону деревянным топориком (ЛКМ — первый угол, ПКМ — второй)
2. Убедись, что выделение целиком внутри твоей защищённой зоны
3. Введи `/ps plot create [название]`

**Ограничения при создании:**
- Выделение должно целиком умещаться в одной твоей защищённой зоне
- Выделение не должно перекрываться с уже существующим участком той же зоны

### `/ps plot delete <название|id>`

Удаляет участок.

### `/ps plot add <название|id> <игрок>`

Даёт игроку доступ к участку.

### `/ps plot kick <название|id> <игрок>`

Лишает игрока доступа к участку. Блокировка действует **даже если игрок добавлен в основную защищённую зону (приват от блока)** — внутри участка он не сможет ломать и ставить блоки, открывать контейнеры, двери и калитки, нажимать кнопки и плиты, взаимодействовать с рамками и атаковать существ.

### `/ps plot kickall <игрок>`

То же самое, что `kick`, но сразу для всех участков текущего мира.

### `/ps plot list`

Показывает все участки в текущем мире: название, к какой зоне относится, координаты углов и список игроков, реально имеющих доступ. Игроки, получившие доступ через основную зону(блок привата), помечаются как `(via parent)`.

### `/ps plot flag <название|id> [interact|chest-access] [allow|deny|none]`

Управляет флагами WorldGuard непосредственно на участке, независимо от флагов родительской зоны.

**Доступные флаги:**

| Флаг | Что разрешает |
|---|---|
| `interact` | Правый клик по блокам: кнопки, рычаги, двери, нажимные плиты, зачарователи, верстаки и т.д. |
| `chest-access` | Открытие контейнеров: сундуки, бочки, шалкеры |

**Значения:** `allow` — разрешить для всех, `deny` — явно запретить, `none` — сбросить (наследуется от родительской зоны).

Без явной установки флаги **всегда наследуются** от родительской зоны — поведение по умолчанию не меняется.

Без аргументов (`/ps plot flag <название>`) показывает флаги, выставленные непосредственно на участке.

**Пример — ферма опыта с общедоступным сундуком:**
```
/ps plot create storage      ← выделить зону вокруг сундука/зачарователя
/ps plot flag storage interact allow       ← все могут нажимать кнопки и т.п.
/ps plot flag storage chest-access allow   ← все могут открывать сундуки
# ломать блоки по-прежнему нельзя — наследуется запрет от основной зоны
```

Флаги не обходят список заблокированных игроков (`/ps plot kick`) — кикнутый игрок не получит доступ, даже если флаг выставлен в `allow`.

## Права доступа к участку

Доступ вычисляется по формуле:

> **(участники основной зоны ∪ участники участка) − заблокированные**

| Ситуация | Доступ к участку | Доступ к остальной зоне |
|---|---|---|
| Добавлен в основную зону (`/ps add`) | ✅ | ✅ |
| Добавлен только в участок (`/ps plot add`) | ✅ | ❌ |
| Добавлен в оба | ✅ | ✅ |
| Заблокирован в участке (`/ps plot kick`) | ❌ — даже если в основной зоне | ✅ остальная территория |
| Удалён из основной зоны (`/ps remove`) | ❌ — автоматически теряет и участки | ❌ |

**Владелец основной защищённой зоны всегда имеет полный доступ ко всем участкам внутри неё.** Команда `kick` на владельца не действует.

### Что происходит при удалении из основной зоны

Когда игрок удаляется из основной зоны командой `/ps remove` или `/ps removeowner`, он автоматически удаляется из всех участков этой зоны. В чате отображается количество затронутых участков.

---

## Вложенные зоны защиты

Если внутри одной защищённой зоны находится другая, система автоматически привязывает новый участок к **наименьшей зоне, которая целиком вмещает выделение**:

```
Зона A (большая)
└── Зона B (средняя, внутри A)
    └── Зона C (маленькая, внутри B)

Выделение внутри C             → участок привязывается к C
Выделение внутри B, но вне C   → участок привязывается к B
Выделение внутри A, но вне B   → участок привязывается к A
Выделение пересекает две зоны  → ошибка, создание отклонено
```

При удалении зоны удаляются только напрямую привязанные к ней участки.

---

## Автоматическая очистка

- **При удалении защищённой зоны** — все её участки удаляются вместе с ней.
- **При старте сервера** — если участок остался без родительской зоны (например, та была удалена командой консоли `/rg remove`), он автоматически удаляется. Количество удалённых участков логируется.

## Конфигурация

В `config.toml` сервера добавляется секция:

```toml
[plot]
    # Стоимость создания участка (требует Vault). 0.0 — бесплатно.
    create_cost = 1500.0
```

mvn clean install -DskipTests
